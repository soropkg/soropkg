# soropkg generate

Generate fully-typed TypeScript client bindings for every package in your `[dependencies]`.

```bash
soropkg generate [flags]
```

---

## What gets generated

For each dependency, soropkg generates a `.ts` file in `.soroban/` containing:

- A **`Client` class** with one method per contract function
- **Full TypeScript types** for all arguments and return values, derived directly from the on-chain contract spec
- A **`ClientOptions` interface** for passing RPC URL, keypair, and network passphrase

---

## Flags

| Flag | Description |
|---|---|
| `--mainnet` | Use mainnet contract addresses (default) |
| `--testnet` | Use testnet contract addresses |
| `--out <dir>` | Output directory (default: `.soroban/`) |
| `--package <name>` | Only generate for a specific package |

---

## Example

```bash
soropkg generate
```

```
  ✓  stellar__token.ts
  ✓  blend_capital__blend.ts
  ✓  blend_capital__backstop.ts

Written to .soroban/
```

## Using the generated client

```typescript
import { Client as TokenClient } from './.soroban/stellar__token'

const client = new TokenClient({
  contractId: 'CDLZFC3SYJYDZT7K67VZ75HPJVIEUVNIXF47ZG2FB2RMQQVU2HHGCYSC',
  networkPassphrase: 'Public Global Stellar Network ; September 2015',
  rpcUrl: 'https://mainnet.stellar.validationcloud.io/v1/xycSmFnKMsLaOUPNGa6qFHNreHmf',
})

// Call a function — fully typed
const balance = await client.balance({ id: 'GABC123...' })
```

---

## File naming

Generated files use the pattern `{org}__{package}.ts`, with `/` replaced by `__`:

| Package name | Generated file |
|---|---|
| `stellar/token` | `stellar__token.ts` |
| `blend-capital/blend` | `blend_capital__blend.ts` |
| `soroswap/router` | `soroswap__router.ts` |

---

<Callout>
  Add `.soroban/` to your `.gitignore` — these files should be generated at build time, not committed to source control.
</Callout>
