# soropkg add

Add a published contract package as a dependency in your `soroban.toml`.

```bash
soropkg add <package> [flags]
```

soropkg resolves the requested semver range against the registry, records the resolved version in your manifest, and fetches the contract spec from the network.

---

## Arguments

| Argument | Required | Description |
|---|---|---|
| `<package>` | Yes | Package name with optional version range, e.g. `blend-capital/blend@^2.0.0` |

---

## Flags

| Flag | Description |
|---|---|
| `--mainnet` | Resolve against mainnet contracts (default) |
| `--testnet` | Resolve against testnet contracts |
| `--futurenet` | Resolve against futurenet contracts |
| `--dev` | Add to `[dev-dependencies]` instead of `[dependencies]` |

---

## Version syntax

soropkg uses standard semver ranges:

```bash
soropkg add stellar/token@1.0.0      # exact version
soropkg add blend-capital/blend@^2   # compatible with 2.x.x
soropkg add soroswap/router@~1.2     # compatible with 1.2.x
soropkg add phoenix/pair             # latest published version
```

---

## Example

```bash
soropkg add blend-capital/blend@^2.0.0
```

```
Resolving blend-capital/blend@^2.0.0...
  ✓  Resolved blend-capital/blend@2.0.1
  ✓  Updated soroban.toml
```

Your manifest is updated:

```toml
[dependencies]
"blend-capital/blend" = "^2.0.0"
```

---

<Callout type="warning">
  After adding dependencies, run [`soropkg generate`](/cli/generate) to regenerate TypeScript clients for the new packages.
</Callout>
