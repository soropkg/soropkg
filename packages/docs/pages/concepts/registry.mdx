# Registry

The soropkg registry is the central index of published Soroban contract packages. It stores package metadata, contract specs, and verified WASM hashes — but **not the WASM binaries themselves**. Those live on-chain.

## Publishing a package

See [`soropkg publish`](/cli/publish) for the full publishing workflow. In summary:

1. Authenticate with your GitHub identity
2. soropkg verifies your deployed WASM matches the declared package version
3. Metadata and contract spec are stored in the registry
4. The package becomes searchable and installable

---

## Package ownership

Package names are namespaced to GitHub identities. `my-org/my-protocol` can only be published by the GitHub user or organisation `my-org`. This prevents name squatting and impersonation.
